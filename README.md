# 온라인 예산정책이슈 전자책 서비스

PDF 자료를 웹 브라우저에서 전자책처럼 열람할 수 있는 순수 정적(static) 웹사이트입니다.
로그인이나 서버 없이 GitHub Pages 같은 정적 호스팅에 그대로 올려서 서비스할 수 있고,
자료 추가·삭제는 로컬 PC에서 관리 도구를 잠깐 켜서 처리한 뒤 GitHub에 push하면 됩니다.

## 구조

```
index.html, styles.css, app.js   → 실제 서비스되는 정적 사이트 (그대로 GitHub Pages에 올리면 됨)
manifest.json                     → 자료 목록 + 목차 (관리 도구가 자동으로 갱신)
toc.json                          → 자료별 목차(제목-쪽번호) 원본 데이터
books/                            → PDF 원본 파일
vendor/pdfjs/                     → PDF 뷰어 라이브러리 (오프라인 내장, CDN 불필요)
server.js, copy-vendor.js         → 로컬 전용 관리 도구 (배포되는 사이트에는 영향 없음)
```

사이트 자체는 `manifest.json`만 읽어서 동작하는 완전한 정적 페이지라 서버가 없어도 문제없이 열람됩니다.

## 로컬에서 자료 관리하기

1. 처음 한 번만 의존성 설치:
   ```powershell
   npm.cmd install
   ```
2. 관리 도구 실행 (`서버실행.lnk` 더블클릭 또는):
   ```powershell
   npm.cmd start
   ```
3. 브라우저에서 `http://localhost:5173` 접속 — 로컬에서 실행 중이면 왼쪽에 업로드 패널이 자동으로 나타납니다.
4. `+ PDF 업로드`로 자료를 추가하거나, 목록의 휴지통 아이콘으로 삭제합니다. 추가/삭제할 때마다 `manifest.json`이 자동으로 갱신됩니다.
5. 자료를 열람 중일 때 왼쪽 목차 패널의 `편집` 버튼을 누르면 목차(제목 + 쪽번호)를 직접 입력할 수 있습니다. PDF 자체에는 목차 메타데이터가 없는 경우가 많아서, 표지 등에 적힌 목차를 보고 그대로 입력하면 됩니다. 저장하면 목차 항목을 클릭해 바로 그 쪽으로 이동할 수 있습니다.
6. 다 정리했으면 서버 창에서 `Ctrl+C`로 종료합니다.

> 로그인 기능은 없습니다. 이 도구는 로컬(127.0.0.1)에서만 동작하고, 실행할 수 있는 사람 = 관리자이기 때문입니다.

## GitHub Pages 등 정적 호스팅에 배포하기

1. 위 방법으로 로컬에서 PDF들을 정리합니다 (`books/`와 `manifest.json`이 최신 상태가 됨).
2. 이 프로젝트 폴더를 GitHub 저장소로 push합니다 (`books/`의 PDF, `toc.json`, `manifest.json`까지 함께 커밋).
   - 개별 PDF가 GitHub 파일 용량 제한(100MB)을 넘지 않는지 확인하세요.
3. 저장소 Settings → Pages에서 배포 브랜치를 `main`, 폴더를 `/ (root)`로 지정합니다.
4. 몇 분 후 발급되는 주소로 접속하면 관리자 패널 없이 열람 전용 화면으로 바로 서비스됩니다.
5. 자료를 추가/삭제할 때마다 로컬에서 다시 관리 도구로 정리 → 커밋 → push하면 사이트가 갱신됩니다.

## 제공 기능

- 정적 사이트 그대로 배포 가능 (서버·로그인 불필요)
- 로컬 관리 도구로 PDF 업로드 / 삭제, `manifest.json` 자동 동기화
- 자료별 목차 직접 입력 및 클릭 이동 (표지에만 목차가 적혀있는 PDF도 지원)
- PDF 자료 목록 자동 표시, 자료명 검색
- 페이지 넘김, 단면 보기 / 양면 보기 전환, 확대 / 축소
- 원본 PDF 다운로드
- PDF 뷰어 라이브러리 내장 제공 (오프라인 동작)
